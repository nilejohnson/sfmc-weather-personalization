data extensions needed for this setup
======================================

3 DEs total. create them in contact builder.


DE 1: Cities
------------
just a reference table, 5 rows

CityID         text 50     PK
CityName       text 100
Country        text 100
Latitude       decimal 18,6
Longitude      decimal 18,6
Timezone       text 50

data:
sydney       | Sydney      | Australia | -33.87 | 151.21  | Australia/Sydney
los_angeles  | Los Angeles | USA       | 34.05  | -118.24 | America/Los_Angeles
reykjavik    | Reykjavik   | Iceland   | 64.15  | -21.94  | Atlantic/Reykjavik
tokyo        | Tokyo       | Japan     | 35.68  | 139.69  | Asia/Tokyo
svalbard     | Svalbard    | Norway    | 78.22  | 15.64   | Arctic/Longyearbyen


DE 2: WeatherData
-----------------
the important one. script writes here daily, email reads from it.
name has to be exactly "WeatherData" (script references it by name)
start empty, automation fills it in.

CityID              text 50    PK
CityName            text 100
FetchDate           date
Day1_Date           date
Day1_Temp           number
Day1_WeatherCode    number
Day1_Precipitation  decimal 10,2
Day2_Temp           number
Day3_Temp           number
Day7_Temp_Avg       number
WeatherCategory     text 50

note: temp fields need to accept negatives (svalbard goes to -25 etc)
WeatherCategory ends up being one of: Sunny, Rainy, Cold, Snowy, Cloudy


DE 3: Subscribers_Demo
----------------------
sendable DE for testing. 1 subscriber per city.
make it sendable: subscriberkey -> subscriber key, emailaddress -> email address

SubscriberKey  text 50       PK
EmailAddress   emailaddress 254
FirstName      text 100
CityID         text 50

data:
TEST001 | test1@example.com | Emma  | sydney
TEST002 | test2@example.com | Mike  | los_angeles
TEST003 | test3@example.com | Bjork | reykjavik
TEST004 | test4@example.com | Yuki  | tokyo
TEST005 | test5@example.com | Olaf  | svalbard

swap the test emails for real ones if you wanna actually send tests.


how they link up
----------------
Subscribers_Demo.CityID -> WeatherData.CityID

email grabs CityID with AttributeValue, then does a Lookup on WeatherData.
that's it.
